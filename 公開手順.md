# はじめての公開ガイド — FXコンパス

サイト運営が初めての人向けに、**何も知らない状態から公開してSEOのスタートを切るまで**を順番に書いた指示書。
上から順にやれば、**所要 約1時間・費用0円**でサイトが公開されます。

## 全体像（最初に1分だけ読む）

- あなたが持っているもの: 完成済みのサイト一式（このフォルダ）。記事・デザイン・SEOタグ・法務ページは全部できている。
- ゴール: インターネット上にサイトを公開し、Googleに「このサイトを検索結果に載せて」と申請した状態。
- お金の流れ: 読者があなたのサイトのFTMOボタン → FTMOで購入 → あなたのFTMO管理画面に報酬（購入額の8〜20%）が貯まる → €50以上で出金。
- **やらなくていいこと**: サーバ契約・ドメイン購入（任意）・プログラミング・決済の実装。何も買いません。

つまずいたら最下部の「よくあるつまずき」を見る。それでも駄目ならClaude Codeに「公開手順のSTEP◯で止まった」と聞けばOK。

## STEP 1: アフィリエイト登録（約30分）— 収益の心臓部

このサイトの収益は「プロップ（集客）＋国内の合法アフィリ（実際に稼ぐ）」の組み合わせです。
登録先は2系統あります。**全部そろわなくても、取れたものから入れていけばOK**（未登録の枠はサイト上「準備中」表示で害なし）。

**(A) FTMO 直アフィリ（プロップの第1指名・集客の入口）**
1. ブラウザで https://ftmo.com/en/affiliate-programme/ を開く
2. 「Become an affiliate」から無料登録（名前・メールアドレス）
3. ダッシュボードのあなた専用リンク（例: `https://trader.ftmo.com/?affiliates=あなたのコード`）をメモ帳にコピー

**(B) ASP 登録（VPS・国内FX/CFD・TradingView＝実際の収益主柱。ここが本命）**
ASP（アフィリエイト・サービス・プロバイダ）は複数の広告主をまとめて扱う仲介サービスです。下記に無料登録し、各案件と提携します。
- **A8.net**（https://www.a8.net/）: DMM FX・DMM CFD・GMOクリック証券・ConoHa VPS 等の定番。まずここ。
- **もしもアフィリエイト**（https://af.moshimo.com/）: ConoHa VPS など（A8と単価を見比べる）。
- **TradingView パートナープログラム**（https://jp.tradingview.com/partner-program/）: チャートツール（継続報酬）。
- （任意）アクセストレード等: コインチェック（暗号資産）など。

> 各ASPで提携が承認されると、案件ごとの「あなた専用リンク」と**実際の報酬単価**が管理画面に出ます。
> リンクをメモ帳にコピーし、単価は OPERATIONS.md の単価メモ（links.js の note）を実数で上書きする材料にします。
> **海外FX業者のアフィリには登録しないでください**（日本居住者向けは法律のリスクがあるため、このサイトは扱いません）。

✅ 完了の目安: FTMOリンク＋ASPで取れた案件のリンクをコピーできた（全部そろわなくてもOK・後でSTEP 7で追加可）

## STEP 2: GitHubアカウントとリポジトリ作成（約10分）— 無料の置き場所

GitHubは「サイトのファイルを無料で置いて公開できる場所」です。

1. https://github.com/ → 「Sign up」で無料アカウント作成（**ここで決めたユーザー名が公開URLの一部になる**）
2. ログイン後、右上の「+」→「New repository」
3. Repository name: `prop-navi` ／ **Public** を選択 ／「Add a README file」に**チェックを入れる** → 「Create repository」

✅ 完了の目安: `github.com/あなたのユーザー名/prop-navi` のページが開けた

## STEP 3: 公開準備スクリプトを実行（約5分）— ここはほぼ全自動

1. Macで「ターミナル」アプリを開く（⌘+スペース →「ターミナル」と入力）
2. 次の2行を1行ずつ貼り付けてEnter:

```bash
cd "/Users/tenten/Claude/Projects/スモビジ/sites/prop-navi"
python3 publish.py
```

3. 質問に答える（GitHubユーザー名 / リポジトリ名はEnter → 続けて**各アフィリリンクを1枠ずつ**聞かれます: FTMO・ConoHa VPS・DMM FX・GMO・DMM CFD・TradingView・コインチェック / note / 運営者名 など）
   - **取れていない枠は空Enterでスキップ**でOK（あとでSTEP 7で追加できます）。アフィリ枠は links.js を見て自動で増減するので、将来枠が増えても同じ操作です。
4. 最後に「✔ dist/ の生成と検証が完了しました」と出れば成功。アフィリ枠ごとに「投入済み/未設定」の一覧が表示されます。`dist` というフォルダができている

> 間違えても何度でも実行し直せます（2回目以降は前回の答えがEnterだけで使えます）。

✅ 完了の目安: 「✔ 完了」表示と `dist` フォルダ

## STEP 4: ファイルをアップロード（約10分）

1. ブラウザで自分のリポジトリ（STEP 2のページ）を開く →「Add file」→「Upload files」
2. Finderで `sites/prop-navi/dist` を開き、**中身を全部選択**（⌘+A）してブラウザの点線枠にドラッグ
   - ⚠ `dist` フォルダごとではなく「**中身**」をドラッグ（index.html や assets が直接見える状態にする）
   - フォルダ（assets/guide/legal/tools）ごとドラッグでき、構造はそのまま保たれます。**ブラウザはChrome推奨**（Safariはフォルダのドラッグに失敗することがある）
3. 緑の「Commit changes」を押す

✅ 完了の目安: リポジトリのページに index.html・assets・guide などが並んでいる

## STEP 5: 公開スイッチを入れる（約5分）

1. リポジトリの「Settings」タブ →左メニュー「Pages」
2. 「Build and deployment」の Source: **Deploy from a branch** ／ Branch: **main** と **/(root)** → 「Save」
3. 1〜3分待ってページを再読み込み →「Your site is live at https://〜」と出る → そのURLを開く

✅ 完了の目安（5点チェック）:
- トップページが白×ブルー〜バイオレットのグラデのデザインで表示される（ヒーローに方位磁針マスコット「コンパスくん」＋カテゴリチップ＋「口座をさがす」ランキングカードが出る）
- メニュー（はじめてのFX／口座比較／ツール・環境／プロップ／用語集 等）から各記事が開ける
- 計算ツールに数字を入れると結果が出る（下部のROIシミュレータも動く）
- 記事ごとに**関連するサービスのボタン**が出て、リンク先が**あなたの紹介リンク**になっている（PR表記付き。例: VPS記事ならConoHa、国内口座記事ならDMM FX等。未登録の枠は「準備中」表示でOK）
- スマホでも開いて崩れていない（比較表は横スクロールできる）

## STEP 6: Googleに登録する（約15分）— SEOのスタート地点

これをやらないと検索結果に載るのが大幅に遅れます。

1. https://search.google.com/search-console を開き、Googleアカウントでログイン
2. プロパティタイプは「**URLプレフィックス**」を選び、あなたの公開URL（`https://ユーザー名.github.io/prop-navi/`）を入力
3. 所有権の確認は「**HTMLファイル**」方式を選ぶ → `google〜.html` というファイルがダウンロードされる
4. そのファイルを **STEP 4と同じ手順**でリポジトリにアップロード（Add file → Upload files → Commit）
5. Search Consoleに戻り「確認」を押す → 「所有権を確認しました」
6. 左メニュー「サイトマップ」→ `sitemap.xml` と入力 → 送信
7. 左上の「URL検査」にトップページのURLを入れ「インデックス登録をリクエスト」（余裕があれば guide/ の記事4本も同様に）

✅ 完了の目安: サイトマップのステータスが「成功しました」

## STEP 7: 後からやること（収益に直結）

- **アフィリリンクを後から取得した場合（FTMO・ConoHa・DMM・GMO・TradingView 等）**: ターミナルで `python3 publish.py` を再実行し、該当の枠の質問でリンクを入力（前回入れた枠はEnterで維持）→ できた `dist` の中身をSTEP 4の手順で再アップロード（同名ファイルは自動で上書き）。**アフィリは取れた枠から順に増やしていけばOK**です。
- **記事を更新した場合（週次更新・追記など）**: 記事のHTMLを直したら、必ず **`python3 publish.py` を再実行** して `dist` を作り直し、**`dist` の中身をSTEP 4の手順で再アップロード**してください（ソースを直しただけでは公開サイトに反映されません。publish.py が dist にURL等を流し込んで初めて公開用になります）。
- **note記事**: サイトの記事を読み物に再編集してnoteに公開（手順は OPERATIONS.md §5。アフィリリンクはnoteに貼らない＝規約违反防止。サイトへ誘導する）→ 公開URLを `publish.py` 再実行で投入 → 再アップロード

## 公開後の運用（週1回・15分）

Claude Codeに「**FXコンパスを週次更新して**」と言うだけ。各社のルール変更や規制動向を反映 → 検証 → `dist` 再生成までやります。あなたは**`dist` の中身を再アップロード（STEP 4）するだけ**。

数字を見る場所:
- **Google Search Console**: 検索に何回出て何回クリックされたか（最初の2〜4週は0でも正常）
- **各アフィリ管理画面**: FTMOダッシュボード＋各ASP（A8.net・もしも・TradingViewパートナー等）のクリック数と報酬。収益の主柱はVPS・国内FX/CFD・チャートツールの国内合法アフィリです

## （任意）収益が出始めたら検討すること

- **独自ドメイン**（年1,000〜1,500円程度）: `github.io` より信頼とSEOに有利。お名前.com等で取得し、GitHub Pages の Custom domain に設定（このときは `publish.py --url https://新ドメイン` で再生成→再アップ）
- 2サイト目の横展開（OPERATIONS.md §6）

## よくあるつまずき

| 症状 | 対処 |
|---|---|
| `python3` が見つからないと言われる | `python3 --version` で確認。出なければApp Storeの案内に従いインストール（Macは大抵入っている） |
| アップロード後にサイトが404 | 5分待つ→ Settings→Pages の設定（main / root）を再確認。`dist`フォルダごとアップしていないか（index.htmlがリポジトリ直下に必要） |
| デザインが崩れて文字だけ表示 | assetsフォルダがアップされていない。STEP 4をやり直し |
| FTMOボタンが「準備中」のまま | リンク未投入。STEP 7の1つ目を実行 |
| Search Consoleで所有権確認に失敗 | `google〜.html` がリポジトリ直下にあるか確認。アップ後1〜2分待ってから「確認」 |
| 検索結果に全然出ない | 公開後2〜4週間は0が普通。Search Consoleの「ページ」でインデックス済みかだけ確認して待つ |
